stackstats.py

The objective of this simple application is to retrieve data from stackoverflow’s API, calculate some simple statistics and show them to the user as HTML or JSON format. Since an OAuth key is not used stackoverflow responses to our requests with a portion of data and a maximum of 300 requests daily.

Installation

In order to run the application, Python 2.7 version at least is needed installed in your machine. To download the latest version go to:

  • For Windows users (https://www.python.org/downloads/windows/)
  • For Mac users (https://www.python.org/downloads/mac-osx/)

  • For other platforms (https://www.python.org/download/other/)

    • After installing Python Python should be linked in System variables or Enviroment variables.

If python is installed correctly after typing python --version in cmd or terminal, the current version of python version should be returned in your screen(e.g below).

Other requirements

Standard python build-in libraries are used and there is no need for more modules to be installed.

Running the application

To run the application open your terminal and use cd command to change to the same directory where the application is lives(e.g below).

Basic Syntax

In order to use stackstats.py correctly a specific syntax has to be followed. Syntax for stackstats.py is stackstats.py --since <date> --until <date> [--output-format JSON or HTML]

until: expects the user to input the starting point date to search in the API. Date must follow the Year-Month-Day format

since: expects the user to input the ending point date to search in the API. Date must follow the Year-Month-Day format

output-format: is optional and it is not necessary to run the application. Expects JSON or HTML format in order to output the result.

Running the tests

4 Tests are implemented to check if the Algorithms in the script are running correctly. Similar data like the responsed data of the API have been used to check the correctness of the algorithms. To run the tests, open your terminal/cmd and change to its directory using cd shell command. Then to compile and execute it, run test_stackstats.py command as shown below. The results to be expected are 4 dots which means that they passed the tests successfully. If any of the tests fails, a ‘F’ letter will be shown.

  1. First test assures us with the number of the correct answers that should be returned
  2. Second one checks the total score of all the answers the data contains.
  3. Third test checks if the top ten answers are returned according to their top score. Also counts their respective comments and the id of the answers.
  4. Last but not least, this test takes 2 different dates that resemble the value thats its passed on the --since and --until arguments. Then it checks if they are converted correctly in UTC epoch time, which also reassures the validity of the data to be returned.
